# Port Inspector

A lightweight SwiftUI menu bar utility for macOS that continuously monitors
listening TCP/UDP ports and shows which process owns each one, in real time.

The menu bar shows a live count of listening services (with a warning glyph when
any are exposed to the network). The dropdown lists each service with its
process, PID, protocol, bound address, and an exposure indicator
(localhost-only vs. reachable from the network), plus search and quick actions.

## Architecture

```
MenuBarExtra (UI)  ─▶  PortStore (@MainActor ObservableObject)  ─▶  PortScanner
   live count            published entries, timer, search             lsof -F parser
```

- **`PortScanner`** — shells out to `lsof -F` for TCP listeners and bound UDP
  sockets, parses the field output, and normalizes it into `PortEntry` values.
  Designed to be swapped for `libproc`/`proc_pidinfo` later.
- **`PortStore`** — owns the 5s scan timer, runs scans off the main actor, and
  publishes sorted/filtered results.
- **`MenuContentView`** — the dropdown: header with counts, search, the service
  list, and a footer.

## Build & run

Two interchangeable build systems share the same `Sources/PortInspector`:

**Xcode (for debugging in the IDE)** — project is generated by
[XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`:

```sh
xcodegen generate            # regenerate after adding/removing source files
open PortInspector.xcodeproj # then ⌘R to run
```

The app target is unsandboxed with a real bundle identifier
(`com.local.portinspector`), so it runs cleanly without the bundle-less-binary
console warnings. **Re-run `xcodegen generate` whenever you add or remove
files** — sources are globbed at generation time.

**SwiftPM (for terminal builds / CI / headless scans):**

```sh
swift build && swift run                 # debug run
swift run PortInspector --scan           # print parsed ports and exit
./Scripts/build-app.sh                   # release → build/PortInspector.app
open build/PortInspector.app             # menu-bar app (no Dock icon)
```

> Do **not** enable App Sandbox — a sandboxed build cannot spawn `lsof` or call
> `kill()`. The Xcode target ships with no entitlements file on purpose.

## Notes & limitations (Phase 1)

- Run as a normal user, `lsof` can't fully inspect root-owned sockets, so some
  system listeners may be missing or sparse. A privileged scan path is future
  work.
- UDP has no "LISTEN" state; unconnected bound UDP sockets are treated as
  listening and connected flows are filtered out.

## Roadmap

- **Phase 1** ✅ — menu-bar app, lsof scanner, searchable list, exposure
  indicator, copy/terminate.
- **Phase 2** ✅ — per-process CPU/memory (`ps`) and true binary path
  (`proc_pidpath`, robust to argv[0] rewriting like ssh); quick actions:
  Reveal Binary in Finder, Open in Browser (http/https heuristic, TCP-only),
  Copy Executable Path.
- **Phase 3** — Docker/Colima container correlation (including ssh VM
  port-forwards).
- **Phase 4** — migrate from `lsof` to `libproc`/`proc_pidinfo`.
